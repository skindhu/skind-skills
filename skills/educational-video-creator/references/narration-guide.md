## Writing Principles

### Core Guidelines

1. **Clarity First**: Every sentence should be immediately understandable
2. **One Idea Per Sentence**: Don't cram multiple concepts together
3. **Active Voice**: "The wing creates lift" not "Lift is created by the wing"
4. **Concrete Examples**: Abstract → Concrete → Abstract

### Audience Adaptation

#### For Children (6-12 years)

```markdown
Language:
- Use simple, everyday vocabulary
- Short sentences (8-12 words average)
- Avoid jargon, or explain it immediately
- Use "you" to create connection

Tone:
- Curious and wondering
- Encouraging and positive
- Like a friendly teacher or older sibling

Examples:
✓ "你有没有试过把手伸出车窗外？"
✗ "空气动力学原理表明..."

✓ "这就像一个神奇的魔法！"
✗ "这是物理定律的必然结果。"
```

#### For Teenagers (13-18 years)

```markdown
Language:
- Can use some technical terms (with brief explanations)
- Medium sentence length (12-18 words)
- Occasional humor is welcome
- Reference current topics/interests

Tone:
- Conversational but informative
- Respect their intelligence
- Connect to real-world applications
```

#### For Adults (General)

```markdown
Language:
- Technical terms acceptable
- Varied sentence structure
- More complex relationships between ideas
- Nuanced explanations

Tone:
- Professional but accessible
- Assumes basic knowledge
- Provides depth and context
```

## Script Structure

### Scene Script Template

```markdown
## Scene [Number]: [Title]

### Narration
[Write the exact words to be spoken]

### Timing Notes
- Total duration: [X] seconds
- Words: [X] (aim for 2.5-3 words/second)
- Pauses: [mark significant pauses with ...]

### Visual Sync Points
- "[Key phrase]" → [Visual action]
- "[Key phrase]" → [Visual action]

### Emphasis
**Bold** = Stressed words
*Italic* = Softer tone
[PAUSE] = Deliberate pause
```

### Example Script

```markdown
## Scene 3: Explaining Lift

### Narration
"现在我们来看第一个力——**升力**。[PAUSE]

升力是让飞机飞起来的*最重要*的力量。
它来自飞机的翅膀。

看这个机翼的形状：[PAUSE]
上面是**弯曲**的，下面比较**平**。

当飞机向前移动时，
空气从机翼的上面和下面*同时*流过。

因为上面弯曲，空气要走**更长**的路，
所以它流得**更快**。

这里有个神奇的规律：
空气流得越快，压力就**越小**。[PAUSE]

所以机翼上面的压力...比下面小。
下面的空气就把机翼往上**推**！

这就是升力的秘密！"

### Timing Notes
- Total duration: 45 seconds
- Words: 约140字 (~3字/秒)
- Major pauses at concept transitions

### Visual Sync Points
- "机翼的形状" → Show wing cross-section
- "上面是弯曲的" → Highlight curved top
- "空气从机翼...流过" → Animate airflow
- "流得更快" → Speed up top airflow animation
- "压力就越小" → Show pressure difference
- "往上推" → Animate lift force arrow

### Emphasis
- "升力" - Key term, emphasize
- "弯曲" / "平" - Contrast
- "更快" / "越小" - Cause and effect
- "推" - Action word
```

## Writing Techniques

### Hook Patterns

**Question Hook**
```
"你有没有想过，为什么[surprising observation]？"
"如果我告诉你[counterintuitive fact]，你会相信吗？"
```

**Scenario Hook**
```
"想象一下，你正在[relatable situation]..."
"闭上眼睛，想象[vivid scene]..."
```

**Challenge Hook**
```
"今天，我们要解开一个谜题：[mystery]"
"有一个问题困扰了人类几千年：[big question]"
```

### Explanation Patterns

**The Analogy Pattern**
```
"[Complex concept] 就像 [familiar thing]。
就像 [analogy explanation]，
[concept] 也是这样工作的。"

Example:
"机翼产生升力就像你的手伸出车窗。
当你把手掌向上倾斜，空气会把你的手往上推。
机翼也是用同样的原理，只是设计得更精确。"
```

**The Build-Up Pattern**
```
"首先，[foundation concept]。
接下来，[building on that]。
最后，[conclusion]。"
```

**The Contrast Pattern**
```
"你可能以为 [common misconception]。
但实际上，[correct understanding]。
区别在于 [key difference]。"
```

### Transition Phrases

**Moving to Next Point**
```
- "现在我们来看..."
- "接下来..."
- "那么...是怎么回事呢？"
- "说到这里，你可能会问..."
```

**Connecting Ideas**
```
- "这就意味着..."
- "因此..."
- "换句话说..."
- "这就是为什么..."
```

**Summarizing**
```
- "所以，总结一下..."
- "简单来说..."
- "记住这一点：..."
- "最重要的是..."
```

## Pacing and Rhythm

### Speech Rate Guidelines

| Audience | Words/Minute | Characters/Second (Chinese) |
|----------|--------------|----------------------------|
| Children | 100-120 | 2.5-3 |
| Teenagers | 120-140 | 3-3.5 |
| Adults | 140-160 | 3.5-4 |

### Pause Types

```
[.]     Short pause (0.3-0.5s) - Between sentences
[..]    Medium pause (0.5-1s)  - Between ideas
[...]   Long pause (1-2s)      - Before important reveal
[BEAT]  Dramatic pause (2-3s)  - For emphasis
```

### Rhythm Patterns

**Building Tension**
```
Short sentence. [.]
Another short one. [.]
Then something a bit longer to develop the idea. [..]
And finally... [BEAT]
The revelation that ties it all together!
```

**Explaining Complex Ideas**
```
[State the concept simply.] [..]
[Expand with detail.] [.]
[Give an example.] [..]
[Restate in different words.] [.]
[Connect to bigger picture.]
```

## Narration-Visual Sync

### Timing Principles

1. **Lead with Visuals**: Show element slightly before mentioning
2. **Match Emphasis**: Stress words when key visual appears
3. **Allow Processing**: Pause after complex visuals
4. **Avoid Overload**: Don't describe everything on screen

### Sync Notation

```markdown
NARRATION                          | VISUAL
-----------------------------------|----------------------------------
"首先来看升力..."                   | Lift arrow appears
[0.5s pause]                       | Arrow settles
"它来自机翼"                        | Zoom to wing
"看这个形状"                        | Wing cross-section appears
[1s pause]                         | Allow viewer to observe
"上面是弯曲的"                      | Top curve highlights
```

### Anti-Patterns

```markdown
✗ DON'T: "You can see the red arrow pointing up"
  (Viewer can already see it)

✓ DO: "This force—lift—pulls the plane upward"
  (Explains meaning, not visual)

✗ DON'T: Narration and visual showing different things

✓ DO: Narration explains what visual demonstrates
```

## Subtitle Guidelines

### Chinese Subtitle Formatting

```markdown
Length:
- Maximum 15 characters per line
- Maximum 2 lines on screen
- Minimum 1.5 seconds display time

Punctuation:
- Use full-width punctuation（，。！？）
- No period at end of subtitle lines
- Use "..." for trailing thoughts

Line Breaks:
- Break at natural phrase boundaries
- Don't break in middle of terms
- Keep subject-verb together when possible
```

### Example Subtitle Breakdown

```
Narration: "升力是让飞机飞起来的最重要的力量。它来自飞机的翅膀。"

Subtitles:
[1] 升力是让飞机飞起来的
[2] 最重要的力量
[3] 它来自飞机的翅膀
```

## Quality Checklist

Before recording/finalizing narration:

- [ ] Read aloud - does it sound natural?
- [ ] Check word count matches timing
- [ ] Verify technical accuracy
- [ ] Ensure audience-appropriate language
- [ ] Mark all pauses and emphasis
- [ ] Sync points with visuals documented
- [ ] No tongue-twisters or awkward phrases
- [ ] Consistent terminology throughout

## TTS 生成注意事项

### 文本预处理

在将字幕文本传给 Edge TTS 之前，需要预处理：

- **移除标记**: 删除 `[PAUSE]`、`[BEAT]`、`[.]`、`[..]`、`[...]` 等停顿标记，用无声间隔（segment 间的 GAP_FRAMES）替代
- **数字转换**: 数字写成中文以确保正确发音：「3」→「三」，「100万」→「一百万」
- **英文缩写**: 加注音提示或写成全称，如 `DNA` → `D-N-A`，`API` → `A-P-I`
- **特殊符号**: 移除或替换不适合朗读的符号（如 `→`、`×`、`÷`）

### 字幕分段原则（影响 TTS）

字幕分段直接决定 TTS 音频的颗粒度，需注意：

- **每段 ≤25 个字**: 保持语义完整，太短会导致过多音频文件，太长会降低同步精度（generate-tts.ts 自动按 ≤25 字分段）
- **在自然停顿处分段**: 在句号、逗号或语义转折处断开
- **避免拆分句子**: 不要把一句话拆到两个段，这会导致 TTS 语调不自然
- **保持节奏感**: 相邻段落长度不宜差异过大

**好的分段:**
```
段1: "你有没有想过"
段2: "飞机是怎么飞起来的"
段3: "今天我们来揭开这个秘密"
```

**不好的分段:**
```
段1: "你有没有"          ← 太短，语义不完整
段2: "想过飞机是怎么"     ← 跨句断开
段3: "飞起来的今天我们来揭开这个秘密"  ← 太长
```

### 语速参考（Edge TTS --rate 参数）

根据目标观众选择合适的语速：

| 观众 | --rate 参数 | 说明 |
|------|------------|------|
| 儿童内容 | `"-10%"` 至 `"-5%"` | 留足理解时间 |
| 青少年 | `"-5%"` 至 `"+0%"` | 自然节奏 |
| 成人 | `"+0%"` 至 `"+5%"` | 稍快但清晰 |

默认推荐使用 `--rate="-10%"` 作为起点，再根据实际时长调整。
